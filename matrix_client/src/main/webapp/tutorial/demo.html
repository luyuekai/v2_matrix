why use scripter to process big data analyzing?

Scripter Data Analyze Process
Whole lifecycle chain

Scripter pictured a whole lifecycle chain for big data analyzing. It includes of metadata editing, data preparing, data importing, data analyzing and data outputing.

Metadata Editing
Service for data

Scripter Metadata is the pre-condition of the data analyzing, it contains the rules of the game.

Table Creation
Easy way for data storage maintainance

Before analyzing, create the data storage first, using Scripter Table creation tutorials to process step by step.

Data Importing
From HDFS System

The Scripter provide an visualizable HDFS browser which can exploring the files in a fantasy way. And importing the files data into data storage with the HDFS file data importing tutorials.

Data Analyzing and...
Visualization, Advanced Iteration Analyzing

In analyze notebook, you can edit and submit script for hive, pig, sparksql, impala etc. Sounds great huh? Try it now! Now that the analyzing job has done, you can go further with the result in scripter. Visualizing them, Transforming them into other format, all is easy in scripter. Not only the result, but all the structral data in hdfs make sense!

How to transfer Data?

1， import generic_algorithm.js file
2， set serverData variable as row data reference
3， invoke DataTransferPOJO.serverJsonData2TableData(serverData) function
The code frage should be like this:
var tableData = DataTransferPOJO.serverJsonData2TableData(serverData);

自定义Table使用方法
  • 1 初始化Table组件
  • 2 绑定Table组件与页面DOM元素
  • 3 设置分页值(可选, default value is 10)
  • 4 加载数据
  • 5 构建视图
  • example:
    • //1 初始化Table组件
      var viewModel = new ListViewModel();
    • //2 绑定Table组件与页面DOM元素
      ko.applyBindings(viewModel, document.getElementById('contentDIV'));
    • //3 设置分页值(可选)
      viewModel.pageMaxSize(20);
    • //4 加载数据
      var tableData = DataTransferPOJO.serverJsonData2TableData(serverData);
      viewModel.buildData(tableData.result);
      viewModel.columnNames(tableData.header);
    • //5 构建视图
      viewModel.buildView();
