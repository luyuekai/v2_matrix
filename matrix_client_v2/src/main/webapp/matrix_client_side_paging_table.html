什么是前端页面分页技术
  • 当数据可以一次从服务端全部返回时，可以对数据直接在前端进行分页显示的技术.
  • 前端分页可以在数据量较大时提供较好的客户视觉体验和性能，因为分页操作无须多次和服务端交互
  • 前端分页一般包含的控件分为：
    • Table
    • Search Input
    • Search Button
    • Paging Total Size Label
    • Paging Components
    • Paging Size Switcher
为什么要制作自定义Table控件
我们在使用table的过程中，发现前端JS table组件或多或少都有一个问题，比如:
  • 1， 不支持跨页选择
  • 2， 不支持跨页操作
  • 3， 和其它js有依赖(比如ng-table,强依赖于angularjs)
  • 4， 删选+选择等组合使用操作多产生bug
  • 5， 从server端拿取数据后的DOM render有问题
  • 6， 不支持跨页全选/反选

How to transfer Data?

1， import generic_algorithm.js file
2， set serverData variable as row data reference
3， invoke DataTransferPOJO.serverJsonData2TableData(serverData) function
The code frage should be like this:
var tableData = DataTransferPOJO.serverJsonData2TableData(serverData);

自定义Table使用方法
  • 1 初始化Table组件
  • 2 绑定Table组件与页面DOM元素
  • 3 设置分页值(可选, default value is 10)
  • 4 加载数据
  • 5 构建视图
  • example:
    • //1 初始化Table组件
      var viewModel = new ListViewModel();
    • //2 绑定Table组件与页面DOM元素
      ko.applyBindings(viewModel, document.getElementById('contentDIV'));
    • //3 设置分页值(可选)
      viewModel.pageMaxSize(20);
    • //4 加载数据
      var tableData = DataTransferPOJO.serverJsonData2TableData(serverData);
      viewModel.buildData(tableData.result);
      viewModel.columnNames(tableData.header);
    • //5 构建视图
      viewModel.buildView();
