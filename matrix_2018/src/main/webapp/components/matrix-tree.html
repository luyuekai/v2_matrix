    树型组件由子树结点和叶子结点构成，最广泛的用例为文件夹和文件

树型结构是一个具有层次关系的N层结构，每一层的本质是一个一维数组，数组中存放着结点数据，下面是树型组件的结点数据结构，请牢记：

  {
    "name": "data",                                //当前结点名称
    "type": "folder",                              //当前结点类型 folder/item
    "path": "data",                                //当前结点路径，可选项
    "attr": {                                      //当前结点树型，可选项
      "id": "data",                                //当前结点标识
      "data-icon": "glyphicon glyphicon-cloud",    //当前结点图标，只对item类型生效
      "style": "color: rebeccapurple;"             //当前结点样式
    }
  }


树型组件具备两种事件：点击事件和展开事件。其中展开事件只针对类型为folder类型的结点。
     信息: 请把展开事件理解为获取下一级子树数据源的操作！

3,编写tree_data_source_listener函数的实现
                      
                        /**
                         * 获取Tree组件数据源监听函数
                         * @param  {[type]}   openedParentData 出发事件的数据：文件夹
                         * @param  {Function} callback         回调函数
                         * @return {[type]}                    无返回值
                         */
                        function tree_data_source_listener(openedParentData, callback)
                      
openedParentData为当前事件出发的结点元素，通过获取它的属性，比如path，再经过具体的业务逻辑，获取到完整的下一层子树完整数据集。
请再次牢记，获取到的数据集是一个标准的存放着结点数据的一维数组。

callback是默认的回调函数，当数据完整获取后，将其传回此函数中，交由Tree组件完成DOM的render操作。

具体细节可以查阅对应的js代码，下面是一个完整的函数实现的例子：
                        
                        function tree_data_source_listener(openedParentData, callback) {

                          var childObjectsArray = retrieve_data_mock(openedParentData);
                          callback({
                            data: childObjectsArray
                          });
                        }
